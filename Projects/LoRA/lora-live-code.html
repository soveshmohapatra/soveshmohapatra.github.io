Deconstructing LoRA

Part 2: Fifty Lines of PyTorch

By Sovesh Mohapatra • May 2026
Overview

The whole machinery is 50 lines: a wrapper class, a recursive injector, helper functions for trainable-parameter bookkeeping, plus a 5-line merge function for zero-overhead inference.

The LoRA Wrapper

class LoRALinear(nn.Module):
    def __init__(self, base: nn.Linear, rank: int = 4, alpha: float = 8.0):
        super().__init__()
        self.base = base
        for p in self.base.parameters():
            p.requires_grad_(False)
        self.A = nn.Parameter(torch.randn(rank, base.in_features) / rank)
        self.B = nn.Parameter(torch.zeros(base.out_features, rank))
        self.scale = alpha / rank

    def forward(self, x):
        return self.base(x) + (x @ self.A.T) @ self.B.T * self.scale

Three things: the base is frozen (no gradients flow into it), $A$ is randomly init / $B$ is zero (initial product is zero), and the forward path keeps the rank-$r$ intermediate to avoid materialising the dense $BA$.

Why the Forward Pass Is Two Matmuls

The naive way — first form $BA$ as a dense $d_\text{out} \times d_\text{in}$ matrix — would cost $O(d_\text{out} \cdot d_\text{in})$ just to construct, defeating the point. Instead, factor through the rank-$r$ intermediate: $A x$ costs $O(r \cdot d_\text{in})$, then $B(Ax)$ costs $O(r \cdot d_\text{out})$. Total $O(r(d_\text{in} + d_\text{out}))$, much smaller than $O(d_\text{in} d_\text{out})$.

The Recursive Injector

def inject_lora(module, target_class=nn.Linear, rank=4, alpha=8.0):
    injected = []
    for name, child in list(module.named_children()):
        if isinstance(child, target_class):
            lora = LoRALinear(child, rank=rank, alpha=alpha)
            setattr(module, name, lora)
            injected.append(lora)
        else:
            injected.extend(inject_lora(child, target_class, rank, alpha))
    return injected

Walks the module tree, replaces every nn.Linear with its LoRA-wrapped version. For Transformers, you typically restrict to attention projections.

Merging at Inference Time

def merge_lora(lora_linear):
    base = lora_linear.base
    delta = lora_linear.scale * (lora_linear.B @ lora_linear.A)
    base.weight.data += delta
    return base

After merging, the model looks structurally identical to the pretrained base. Inference latency is the same.

Usage

base_model = load_pretrained()
freeze_all(base_model)
inject_lora(base_model, rank=4, alpha=8)
opt = torch.optim.AdamW(p for p in base_model.parameters() if p.requires_grad)
train(base_model, target_data)
