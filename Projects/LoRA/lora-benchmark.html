Deconstructing LoRA

Part 3: 4.59% Beats Full Fine-Tune

By Sovesh Mohapatra • May 2026
Overview

Pretrain a 4-layer MLP on moons. Define a target task: the same moons rotated by 45°. Adapt with full fine-tuning vs LoRA at ranks 2/4/8.

Setup

Base task: two-moons, $n=1000$, noise $0.15$. Target task: same data rotated $45^\circ$. Architecture: 4-layer MLP, hidden $=128$, total params $33{,}666$. Zero-shot transfer of the pretrained model to the rotated task: $63.2\%$.

Results

MethodTrainable params% of full FTFinal acc
From scratch$33{,}666$$100.00\%$$98.90\%$
Full fine-tune$33{,}666$$100.00\%$$99.00\%$
LoRA ($r = 2$)$1{,}544$$4.59\%$$99.10\%$
LoRA ($r = 4$)$3{,}088$$9.17\%$$99.10\%$
LoRA ($r = 8$)$6{,}176$$18.34\%$$98.90\%$

LoRA at rank $2$ — $1{,}544$ trainable parameters — matches or marginally beats full fine-tuning. The other $32{,}122$ parameters remain frozen.

Why $r=8$ Does Not Outperform $r=2$

The intrinsic rank of the useful weight change for this task is small. Adding capacity beyond it buys variance, not signal — when the true effect is low-dimensional, increasing rank adds noise without adding usable degrees of freedom. Rank is a hyperparameter you tune from below; rank $> 16$ is rarely useful short of major distribution shift.

Practical Implications

Memory: only LoRA params need optimizer state. With rank-8 LoRA on a 7B model, AdamW state shrinks from $\sim 56$ GB to $\sim 60$ MB. That is the difference between needing an A100 and fitting on a $24$ GB consumer GPU.

Serving: ship one frozen base + many small LoRA adapters. Switching tasks is a parameter-load, not a model-reload.

Composition: LoRA adapters are linear. Sum or interpolate two LoRAs trained on different tasks for stacked behaviour.

Reversibility: the base is untouched. Undo a fine-tune by removing the adapter.

Why Freezing Matters

Freezing is not memory optimisation — it is what makes adapter-style fine-tuning work in production. Drift the base and you lose the ship-one-base property, the catastrophic-forgetting guarantee, and adapter composability.
