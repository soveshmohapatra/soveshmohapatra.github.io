Ground-Truth FHN

A_PARAM, B_PARAM, TAU = 0.7, 0.8, 12.5

def fhn_rhs(state, I):
    v, w = state
    dv = v - (v ** 3) / 3.0 - w + I
    dw = (v + A_PARAM - B_PARAM * w) / TAU
    return np.array([dv, dw])

def rk4_step(rhs, state, I, dt):
    k1 = rhs(state, I)
    k2 = rhs(state + 0.5 * dt * k1, I)
    k3 = rhs(state + 0.5 * dt * k2, I)
    k4 = rhs(state + dt * k3, I)
    return state + (dt / 6.0) * (k1 + 2 * k2 + 2 * k3 + k4)

40 trajectories of 80 time-steps at $\Delta t = 0.5$. Half spiking ($I = 0.5$), half quiescent ($I = 0.2$). Random initial states.

The Learnable Dynamics

class ODEFunc(nn.Module):
    def __init__(self, state_dim=2, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim + 1, hidden), nn.Tanh(),
            nn.Linear(hidden, hidden), nn.Tanh(),
            nn.Linear(hidden, state_dim))
        nn.init.zeros_(self.net[-1].weight)
        nn.init.zeros_(self.net[-1].bias)

    def forward(self, t, y, I):
        B = y.shape[0]
        I_vec = I.view(B, 1)
        inp = torch.cat([y, I_vec], dim=-1)
        return self.net(inp)

Critical design choice: $t$ is not in the input. This matches FHN's autonomous structure and is what lets the model extrapolate past the training horizon without leaving its training distribution of inputs.

RK4 Integrator

def rk4_integrate(func, y0, I, t_eval):
    states = [y0]
    y = y0
    for i in range(len(t_eval) - 1):
        t = t_eval[i]
        dt = t_eval[i + 1] - t_eval[i]
        k1 = func(t,           y,                 I)
        k2 = func(t + dt / 2,  y + 0.5 * dt * k1, I)
        k3 = func(t + dt / 2,  y + 0.5 * dt * k2, I)
        k4 = func(t + dt,      y + dt * k3,       I)
        y = y + (dt / 6.0) * (k1 + 2 * k2 + 2 * k3 + k4)
        states.append(y)
    return torch.stack(states, dim=0)

Backprop flows through every k1/k2/k3/k4 evaluation.

Discrete RNN Baseline

def rollout(self, y0, I, n_steps):
    y = y0; states = [y0]
    for _ in range(n_steps):
        inp = torch.cat([y, I.view(-1, 1)], dim=-1)
        y = y + self.net(inp)
        states.append(y)
    return torch.stack(states, dim=0)

Same architecture, same parameter count. The only structural difference is that the RNN's update has no $\Delta t$ scaling — it implicitly bakes whatever $\Delta t$ it trained on into its weights.
